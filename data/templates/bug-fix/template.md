# Problem Statement

Describe the bug clearly and concisely. Include:
- What is happening (actual behavior)
- What should happen (expected behavior)
- Steps to reproduce
- Error messages or logs

## User Perspective

Who is affected by this bug? Consider:
- Which users encounter this issue
- How does it impact their workflow
- What workarounds (if any) are they using
- How urgent is the fix for them

## Engineering Framing

Technical analysis of the bug:
- Root cause hypothesis
- Affected components/systems
- Non-goals (what we're explicitly NOT fixing)
- Technical constraints

## Validation Thinking

How will we know the fix works?
- Test cases to verify the fix
- Edge cases to consider
- Regression risks
- Rollback plan
